안에 들어갈 알고리즘 · 포맷

ESP32 / Pi
JSON 메시지 envelope

모든 노드 공통. payload만 노드별 자유.

{
  "node": "tof",         // "tof" | "mmwave"
  "ts":   1717200000123, // ms (Pi 기준 보정)
  "payload": { ... }
}

시간 동기화: ESP32 부팅 시 SNTP 1회 → 이후 millis() + 오프셋.

mmWave
TLV 패킷 검출

TI mmWave SDK OOB Demo 출력 구조:

Magic: 02 01 04 03 06 05 08 07
Header: version, totalPacketLen, ...
TLVs:   type, length, payload ...

1주차는 매직워드 동기화까지. 본 파싱은 2주차.

ToF
4×4 zone moving average

16개 zone 각각 5-tap 이동평균 후 중앙값.

for each zone z:
  avg[z] = mean(last 5 of z)
distance = median(avg[0..15])

median으로 튀는 zone 영향 차단.

ToF
상태 머신 (FSM)
if   dist < 40 cm   → 누움
elif dist < 100 cm  → 앉음
elif dist ≥ 100 cm
     for 1.0s 이상  → 이탈 ◆

상태 전이 0.5초 디바운스로 채터링 차단.

// Hampel (window=7, k=3)
med = median(w)
mad = median(|x - med|)
if |x[i]-med| > k·1.4826·mad
   → x[i] = med
mmWave
보행 특징 추출

TI SDK tracker 출력: id, x, y, vx, vy

// 가장 가까운 1명만
target = argmin √(x² + y²)

// 1초 윈도우 (50ms × 20)
speed = mean(√(vx²+vy²))
sway  = std(x)
hdg   = atan2(mean(vy),
              mean(vx))

파싱 실패 패킷은 무시 (drop count 로그).

ESP32 / Pi
CSV 자동 저장

Pi 5에서 MQTT 구독 → 일별 파일 append.

ts, node, key, value
…123, tof,    state,    sitting
…150, tof,    distance, 87.4
…200, mmwave, speed,    1.12
…200, mmwave, sway,     0.18

flat key-value로 저장 → pandas 분석 편함.

Week 3

합쳐서 위험도 한 글자로

트리거(ToF) → 본 분석(mmWave) → 종합 판정(Pi). 개인별 베이스라인으로 평소와 다른지를 본다.

이번 주 끝나면 — 화면에 "지금 위험도: 주의" 같은 한 줄이 뜬다. ToF가 "이탈" 찍히면 1초 뒤 mmWave 데이터를 같이 보고 종합 판단.
박형석 · ESP32 + Pi
위험도 계산 + 4단계
  • 위험도 모듈 (Pi 5, Python)
  • 가중합으로 단순 시작
  • 분류: 정상 / 주의 / 경고 / 위험
  • 대시보드에 큰 글씨 표시
  • 이력 로그 (SQLite)
유현기 · mmWave
z-score 베이스라인
  • 최근 30회 보행 측정값 저장
  • SQLite에 user별 mean / std 누적
  • z = (x − mean) / std
  • 2σ 이상 이탈 시 위험도↑
  • 30회 미만은 고정 임계값 폴백
조우진 · ToF
이탈 임박 단계 추가
  • 8×8 zone 무게중심(centroid)
  • centroid가 가장자리 방향 이동
  • 가장자리 zone dD/dt 감시
  • 임계 초과 시 "이탈 임박"
  • FSM 4상태로 확장

안에 들어갈 알고리즘

Pi 5 계산
위험도 가중합 + 분류
trigger = (tof_state in
           ["이탈","이탈임박"])
if not trigger: return "정상"

mm_risk = normalize(mm_features)
score   = 0.7·mm_risk
        + 0.3·tof_confidence

if   score < 0.3: return "주의"
elif score < 0.6: return "경고"
else            : return "위험"

가중치는 4주차에 튜닝.

mmWave
z-score 정규화

개인별 베이스라인을 SQLite에 누적.

CREATE TABLE gait_baseline (
  user_id  TEXT,
  feature  TEXT,  -- speed|sway
  values   BLOB   -- ring(30)
);

z = (x_today - mean) / std

mm_risk = sigmoid(-z_speed
                  + z_sway)

속도↓, 흔들림↑ 일수록 위험.

ToF
centroid 추적
// 무게중심
cx = Σ(x_i·w_i) / Σ w_i
cy = Σ(y_i·w_i) / Σ w_i
  w_i = 1 if zone 유효

// 가장자리 변화율
edge_dD =
  (D_edge[t] - D_edge[t-3])/0.3s

if edge_dD > 30 mm/s
   AND cx > threshold_x
   → "이탈 임박"

ESP32 · 통신 WiFi + MQTT — 표준적
⚠ 함정

  • MQTT QoS — 0이면 손실 가능, 1은 ACK 대기로 느려짐 (보통 0 충분)
  • 시간 동기화: NTP 라이브러리 무거움 → SNTP가 ESP32 내장이라 가벼움
  • Pi 5 IP 자동 찾기: mDNS(pi.local) 또는 고정 IP
