시계열(전이) 접근 — 논문 분석과 우리 진행 방법

목표 = 뒤척임 → 앉음 → 일어섬 → 침대 이탈 이라는 "움직임(동사)" 을 잡는 것. 정지 분류(명사)로는 부족 → 시간축이 필요. 우리 칩(IWR6843)으로 그게 되는지 확인한 두 논문을 분석한다.
1 · 근거 논문 우리 칩으로 "시간축 + 전이"를 실제로 한 두 편
둘 다 프레임을 시간으로 묶어 움직임을 인식한다. 방식이 서로 달라서 — 하나는 점군을 직접, 하나는 격자+순환신경망.
논문 ①
3D 점군 시공간 정보 기반 움직임 인식 (PNHM) Human Movement Recognition Based on 3D Point Cloud Spatiotemporal Information · Sensors 2023 (PMC10708869)
평균 89.5%
쉽게: "사진 한 장"이 아니라 "30장을 이어붙인 짧은 영상" 을 통째로 신경망에 넣는다. 점 구름을 그대로(격자로 안 뭉개고) 처리해서, 희소한 점에 강한 게 강점.
점군 프레임0~64점/프레임→ 30프레임 윈도우≈짧은 영상→ PointNet++ 계층추출FPS 샘플링·MSG→ 프레임 max-pool시간 통합→ 동작 4종걷기·서→앉·서→스쿼트·낙상
하드웨어
IWR6843ISK · 3TX/4RX · 60GHz 우리 칩 계열
전이 포함
있음 — 서→앉, 서→스쿼트 (움직임 자체를 분류)
결과
걷기 94 · 앉기 87 · 스쿼트 84 · 낙상 93 → 평균 89.5% (8명, 900셋, 거리 0.5~2.5m)
시간모델
순환(LSTM) 아님 → 30프레임 max-pool로 시간 통합
논문 ②
재활 환자 모니터링 — 3D CNN + Bi-LSTM Human Activity Recognition for Health Rehabilitation Using mmWave Radar 3D Point Cloud · JEEEMI 2025
Bi-LSTM 90%
쉽게: 점 구름을 격자(voxel)로 바꿔 "3D 사진" 을 만들고(3D CNN이 모양 읽음), 그 사진들의 시간 흐름을 Bi-LSTM(앞뒤 다 보는 기억망) 이 이어 읽는다. 여러 모델을 겨뤄 Bi-LSTM이 이김.
점군18.18 Hz→ 복셀화3D 격자→ 3D CNN공간 특징→ Bi-LSTM앞·뒤 시간맥락→ 활동 6종앉·서·걷·뛰·스쿼트·빈공간
하드웨어
IWR6843AOP · 단일칩 · 60GHz 우리 정확히 그 변형
비교
3D CNN · CNN+LSTM · CNN+Bi-LSTM(최고 90%) · PointNet · PointNet++ · RNN
대상
수술 후 환자 모니터링 (활동 상태 위주 · 전이 라벨은 약함)
주의
데이터셋 미공개 → 재현 어려움
2 · 장단점 두 방식 비교 — 우리한테 뭐가 맞나
구분① PNHM (PointNet++ + 시간창)② 3D CNN + Bi-LSTM
점 처리 점군 직접 → 희소점 손실 없음 (우리 4RX에 유리) 복셀화 → 희소하면 빈 격자 많아 손실 위험
시간 이해 max-pool → 긴 시간 의존성 약함 Bi-LSTM → 앞뒤 맥락 다 봄 (전이에 강함)
전이 검증 서→앉 등 전이 직접 분류·검증 활동 상태 위주 (전이 라벨 약함)
무게 중간 3D CNN 무거움 (Pi 부담)
우리 칩 IWR6843ISK 계열 일치 IWR6843AOP 정확 일치
재현성 방법·데이터 공개적 데이터 미공개
+ 보너스 후보 — mPCT-LSTM: 희소 mmWave 전용 설계 · PCT+LSTM · 97.5% · 0.09 GFLOPS(초경량 → Pi/엣지 적합). "점군 직접(①) + 시간순환(②) + 경량"의 장점을 합친 형태라, 나중에 b단계 1순위 후보.
⚠ 두 논문 모두 부딪힌 물리 한계 (b도 마법 아님)
  • 거리: PNHM 최적 0.5~2.5m, 그 너머 성능 하락 → 우리 "먼 거리" 조건에서 자세 정밀도 급락.
  • 비슷한 자세 혼동: "앉기 vs 스쿼트는 속도·높이가 비슷해 구분 저하" → 우리 누움 vs 걸터앉음 혼동과 동일.
  • 둘 다 라벨된 학습 데이터로 훈련 → 우리는 지금 0개. 그래서 b는 나중, 데이터부터.
3 · 진행 방법 a규칙 → 데이터 축적 → b시계열
b(시계열)가 우리 칩으로 된다는 건 위에서 증명됐다. 단 라벨 데이터가 없어서 바로 못 간다. 그래서 a를 먼저 — a가 곧 b의 연료(라벨)를 만든다.
A
규칙 상태기계
지금 · 데이터0
매 순간 자세(posZ 높이임계 or PointNet) + ROI 게이팅(침대 안/가장자리/밖) + 중심점 궤적(posX/Y) 을 규칙으로 엮어 이탈 전조·이탈·낙상 판정.
✔ 데이터 0으로도 지금 동작 · ✔ 돌리는 동안 30프레임 윈도우+라벨이 자동 축적 → 이게 b의 학습셋.
0
첫 측정 실험
바로 다음
A를 짜기 전에 우리 실제 거리에서 데이터 품질부터 확인: send_mmw.py --no-send → ① 프레임당 점 개수 ② posZ가 눕/앉/서로 갈리나 ③ 이탈 시 posX/Y 궤적 ④ 낙상 height_drop.
→ 이 결과가 "먼 거리에서 자세분류까지 갈지, ROI+궤적만 갈지" 를 정한다.
B
시계열 모델
데이터 쌓이면
A가 모은 라벨로 시계열 모델 학습·이식. 우선순위 → 1) mPCT-LSTM(경량·희소특화·Pi 적합) · 2) PNHM(점군직접·전이검증) · 3) 3D CNN+BiLSTM(전이 맥락 강함).
A의 30프레임 윈도우·전이 정의가 B의 입력으로 그대로 이어짐 → 버리는 작업 없음.
한 줄 결정 — A(규칙, 지금) 로 이탈 감지를 돌리며 라벨을 쌓고, 데이터가 충분해지면 B(시계열: mPCT-LSTM/PNHM) 로 전이를 정밀화한다. A는 임시방편이 아니라 B로 가는 유일한 다리 — 돌려야 연료가 생기니까. 다음 행동 = 「0. 첫 측정 실험」 으로 우리 거리의 데이터 품질부터 눈으로 본다.
