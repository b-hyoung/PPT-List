침대 이탈 감지/예측을 다들 어떻게 풀었나, mmWave는 어디에 쓰이고 무슨 문제가 있나, 그래서 우리가 오늘 확인한 것. (⚠️ 상당수 검색 스니펫 기준 — 원문 대조는 별도 표기)

A 기존 해결방안 — 어떻게 했나 & 한계

방법어떻게 (방식)한계 (단점)
카메라·깊이영상/3D CNN — 몸통 회전·구르기 등 pre-exit 동작 감지 (~87.7%)프라이버시 침해 · 각도·이불에 취약 · 야간/사각
웨어러블(IMU)착용 가속도계로 충격/움직임착용 거부·분실·충전 (치매환자 빼버림) · 단독 ~68%
압력매트·로드셀침대다리 하중 4개 → 무게중심(CoG)이 영역 벗어나면 이탈 · 매트 압력 1D CNN(~93.5%)설치 부담 · 자세 거침 · 오탐 많음 · 사후 위주
진동센서침대 진동으로 앉음 등 자세거칠고 행동 제한적
레이더(mmWave/4D)점군 + 도플러(속도) → 누움→앉음→가장자리→이탈 단계 감지 (Vayyar) · 낙상 97~99%세밀 자세 약함(희소점) · 상용 독점·구독료
마주봄ToF(위치) + mmWave(속도·동역학) 융합, 저가 비접촉데이터 축적 단계 (아래 C)
공통 빈자리 — 다들 "저가 + 비접촉 + 프라이버시 보존 + 오탐 낮음"을 동시에 못 채움. 카메라=프라이버시, 웨어러블=착용, 매트=설치, 레이더=독점. → 우리 자리 = 저가 비접촉 융합.

A+ 상용/배포 사례 (이탈 전조까지)

  • Vayyar Care — 4D mmWave, 누움→앉음→가장자리→이탈 단계별 경보 + 도플러(속도). 영국 Essex 316대. 앉는 순간 즉시 알림.
  • 사이렌케어(국내) — Vayyar 레이더+소형카메라. 인천YWCA요양원. 낙상·침대이탈·호흡·심박.
  • Hikvision — mmWave 레이더, "이탈 전조현상만 집중, 뒤척임 제거". 화장실 overstay 알람.
  • VSTAlert(VirtuSense) — 이탈 30~65초 전 의도 예측 (ML). (스니펫 미검증)
  • SafelyYou — AI 카메라, 낙상 40%↓·ER 80%↓. 근데 카메라 사각(화장실 X).
⚠️ 오탐 현실 — 좋은 시스템도 nuisance(헛알람) ~31%, PPV 68% 수준. "뒤척임 vs 진짜 이탈" 구분 = 승부처이자 평가 지표. "우리가 최초" 금지 — 이미 상용화됨. 우리 = 저가·융합·오탐 감소 실증.

A2 상용화된 것들의 문제 (제품 자체의 한계)

문제내용 (어느 제품)
오탐·알람 피로가장 큰 문제. 좋은 시스템도 nuisance ~31%, 표준 침대알람 정확도 15% → 간호사가 무시하거나 꺼버림(alarm fatigue). (bed-exit alarm effectiveness)
비용·구독료Vayyar·Hikvision = 독점장비 + 월 구독. 저예산 시설 접근 어려움. Vayyar 69베드에 138센서(2/베드) → 방마다 비용↑.
카메라 프라이버시SafelyYou·사이렌케어(카메라 옵션)는 촬영. 화장실 등 못 감(SafelyYou 명시 — 낙상 위험 큰 곳). 거주자·가족 거부감.
설치·거리 제약사이렌케어 1m 이내, Hikvision 벽 2m 중앙·무장애물. 방마다 센서 필요 → 확장 비용.
가림(occlusion) 미탐다인·이불·가구로 가리면 놓침. 레이더 낙상 논문: "인원↑ → 가림 → 미탐(FN)↑".
개인화 부담임계 튜닝 수작업. 민감하면 오탐, 느슨하면 미탐 — 스태프가 환자별 조정해야.
감지 vs 예측(늦음)다수가 사후 감지. 이미 서 있은 뒤 알림 = 대응시간 부족. 예측형도 리드타임 제한.
블랙박스·벤더 락인독점 알고리즘 → 검증·커스텀 불가, 시스템 통합·종속 문제.
효과 근거 논쟁침대알람 단독으론 낙상을 못 줄인다는 연구 다수(스태프 대응이 있어야 효과). 기술만으론 부족.
우리 포지션 — 이 문제들 중 비용(저가 오픈HW)·프라이버시(무카메라)·블랙박스(오픈)·오탐(융합으로 뒤척임 구분) 을 겨냥. "상용을 이긴다"가 아니라 "상용이 못 채운 자리(저가·무카메라·융합·검증가능)" 를 실증.

A3 상용 mmWave(레이더) 제품의 문제 — 콕 집어

Vayyar · Hikvision · 사이렌케어 등 레이더 기반 상용이 실제로 안고 있는 것. (우리가 겪은 물리 한계와 동일 — 그들도 엔지니어링으로 우회할 뿐)

상용 레이더 문제내용
가림·다인 미탐다인·이불·가구로 가리면 놓침. 레이더 낙상 연구 "인원↑ → 가림 → 미탐(FN)↑". 다인 병실에 취약.
설치 위치·각도 민감Hikvision 벽 2m 중앙·전방 무장애물, 사이렌케어 1m 이내. 각도·위치 틀어지면 오작동. 방마다 세팅 필요.
방마다 센서 = 커버리지 비용레이더 FoV 한정 → Vayyar 69베드에 138센서(2/베드). 넓은 병동일수록 센서 수·비용 급증.
뒤척임 오탐 (실재)레이더도 뒤척임을 이탈로 착각 → Hikvision이 "뒤척임·가벼운 움직임 제거"에 애씀(=이 오탐이 상용에도 실재한다는 방증). 세밀 자세는 물리상 약함.
단일 레이더 한계대부분 레이더 단독 → 위치 정밀도 약함. 사이렌케어는 카메라를 덧붙여 보완(→ 프라이버시 문제 다시).
정지자·미세움직임 의존도플러 기반 → 완전 정지면 약함(호흡 미세움직임으로 겨우 유지). 아주 조용한 취침 시 신뢰↓.
환경 반사 노이즈금속·가구 반사, 다중경로 → 오검출·유령 표적.
거리-해상도 트레이드오프멀수록 점·해상도↓ → 넓은 커버리지 vs 정밀도 상충.
핵심 — 상용 레이더도 우리와 같은 물리 한계(가림·뒤척임 오탐·정지자·설치민감)를 가짐. 다만 비싼 하드웨어·독점 튜닝으로 우회. → 우리 = 같은 한계를 저가·ToF융합으로 다루는 게 요점(뒤척임 오탐은 ToF 위치, 가림·침대밖은 상보로).

B mmWave — 어디에 쓰이나 & 어떤 문제

쓰이는 곳 : 낙상 감지(97~99%) · 자세 분류 · 침대 이탈/재실 · 생체(호흡·미세움직임) · 보행 분석. 비접촉·야간·이불 투과.

문제내용 (근거)
희소 점군프레임당 점 적음(우리 ~76, 근데 자세엔 부족). 먼 거리일수록 급감.
세밀 자세 어려움Sensor-Stack Limits(IWR6843AOP, 우리 칩): 4자세 0.674 · 좌/우 눕기 35~42% 혼동 · 엎드림(prone) 배포불가. ✅검증
근본 원인"분류기가 아니라 센서 표현(CFAR 희소점)이 벽. 다음은 raw range-FFT 필요." (같은 논문 결론)
장착 민감우리 대각선 2m 장착 → 좌표축 기울어 "높이"가 왜곡. 재장착(1자)로 개선됨.
속도 노이즈점 doppler는 방사방향·희소라 지저분. track velZ(칼만, TLV 1010)를 써야 깨끗.

B2 실전 문제·버그 (센서별 — 실제 겪은 것)

일반 한계 말고, 돌리면서 실제로 터진 문제들. (팀 깃로그 + 우리 실측)

mmWave 문제내용 / 원인
뒤척임을 이탈로 오탐움직임은 보이나 "뒤척임 vs 나가려는 것" 구분 못 함 → 헛알람. 핵심 미해결(상용 Hikvision도 이걸로 고생).
USB 연결 불안정DATA 자주 멈춤·드롭(Device not configured). cfg 재전송으로 복구. 이전 프로세스 안 죽이면 "multiple access" 포트 충돌.
정지한 사람 놓침cfg 동적 클러터 제거 → 가만히 자는 사람은 점 급감(호흡 미세움직임만). 자는 사람 미탐 위험.
좌표축 기울어짐대각선 2m 장착 → z가 진짜 높이 아님(누운 몸 z=-0.12). 1자(수직) 재장착으로 개선.
empty인데 사람 있음이탈 후 침대 비면 ToF=empty인데 mmWave는 걷는 사람 계속 봄(41점) → 라벨 혼동. (단, 이건 융합엔 장점 — ToF 사각 커버)
다인·사물 혼동여러 명·가구 반사 → 유령 track. nan/inf 쓰레기 슬롯 필터 + 침대 소속 게이트로 환자 특정 필요.
거리 멀면 급감점 수 1/r⁴로 감소 → 먼 거리 자세 붕괴.
회전/왕복 보행 왜곡돌거나 왔다갔다 하면 gait 지표 왜곡 → 직진성 게이트로 배제.
ToF 문제내용 / 원인 (팀이 고친 것)
재실 판정 깜빡임존 값 튀어 재실 on/off 깜빡 → 래치·hold·CONFIRM_EXIT 24프레임(~2.4초)으로 억제(그래서 이탈 확정 2.4초 지연).
누운 사람 놓침수평 배치서 재실 임계 4면 누운 사람 미감지 → 2로 낮춤.
이불이 몸 가림ToF는 표면(이불)만 봄 → 이불 언덕 vs 사람 애매.
옆 통행 오탐침대 옆 지나가는 사람 감지 → 좌우 옆경계 컷.
침대 밖 사각가장자리 넘어가면 놓침(좁은 FoV). 사거리 컷오프 조정(몸통 존 잘림 방지). → mmWave가 커버.
잔상(stale)hold 값 커서 사람 없는데 계속 그림 → hold 대폭↓.
속도·동역학 없음도플러 없어 "뒤척임 속도 vs 이탈 속도" 구분 불가 → mmWave가 보완.
posture=정지자세만ToF posture엔 "이탈(exit)" 없음 → in_bed로 잡아야. (우리가 헷갈렸던 부분)
핵심 상호보완 — mmWave 오탐(뒤척임)·정지자 놓침 ↔ ToF 침대밖 사각·속도없음. ToF(위치)+mmWave(속도)로 서로의 버그를 덮는 게 융합의 실전 이유.

C 오늘 우리가 확인한 것 (실측)

  • 정적 자세 = 높이가 답, 속도 무의미 — sitting vs 뒤척임 높이만 80.3% = 높이+속도 80.3%. doppler 속도는 중요특징 밖.
  • 왜 안 됐나(진단, 데이터로 확인) — ① 뭉갬(시간순 안 씀) ② 정적 자세 라벨(시간 신호 없음).
  • 제대로 하니 됨 — 전이 이벤트 라벨(이탈임박 vs 뒤척임) + 시간순 LSTM + 누수방지 시간분할 → LSTM F1 42 vs RF(뭉갬) F1 13. 시간순이 이탈 3~5배 잘 잡음.
  • 단, 성능 약함 — 이탈 36%만 잡음. 1명·이탈이벤트 17개·proxy 라벨 → 낙관적. 데이터 대량 필요(여러명·100+ 이벤트).
결론 — 방향 : 이탈 예측은 높이+속도+위치 → 단계 → 시간순(LSTM). 라벨은 ToF in_bed 이탈 이벤트 기준. 평가는 F1/오탐(nuisance), ToF-단독 대비 (accuracy는 불균형이라 함정). mmWave 역할 = ToF가 못 보는 동역학·침대 밖 + 교차검증 오탐↓.
